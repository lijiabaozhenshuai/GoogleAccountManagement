# 谷歌账号管理系统 - 打包工具

## 📦 目录说明

```
package/                    # 打包工具目录
├── build_config.py        # 打包配置
├── build.spec            # PyInstaller 规格文件
├── runtime_hook.py       # 运行时钩子
├── 打包.bat              # 打包工具（主脚本）
├── 使用说明.txt          # 使用说明
└── README.md            # 本文件
```

## 🚀 快速开始

### 开发者

1. 进入 `package` 目录
2. 双击 `打包.bat`
3. 选择对应功能

### 功能说明

- **[1] 安装依赖**: 安装所有 Python 依赖包
- **[2] 启动程序**: 以开发模式启动（测试用）
- **[3] 打包成 EXE**: 打包成可执行文件
- **[4] 制作部署包**: 生成给用户的完整部署包

## 📝 打包流程

```
package/打包.bat -> [1] 安装依赖 -> [2] 测试运行 -> [3] 打包 -> [4] 制作部署包
```

## 📂 输出目录

打包完成后会在项目根目录生成:
- `dist/谷歌账号管理系统/` - 打包后的程序
- `谷歌账号管理系统_部署包/` - 用户部署包
- `谷歌账号管理系统_部署包.zip` - 压缩包

## ⚙️ 自定义配置

### 修改打包配置

编辑 `build_config.py`:
- `APP_NAME`: 应用名称
- `datas`: 需要打包的数据文件
- `hiddenimports`: 隐藏导入的模块
- `excludes`: 排除的模块（减小体积）

### 添加隐藏导入

如果打包后提示缺少模块，在 `build_config.py` 中添加:

```python
hiddenimports = [
    'your_module_name',
    # ...
]
```

## 🔍 临时文件管理

运行时临时文件存储在:
- `app_data/` - 应用数据
- `instance/` - 数据库文件

## 💡 提示

- 确保在项目根目录有 `config.json.example`
- 打包前先测试程序能否正常运行
- 第一次打包可能需要几分钟
